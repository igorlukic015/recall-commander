# Recall Commander

## Project Vision

Recall Commander is a **Markdown-first knowledge assessment platform** designed to evaluate understanding rather than simply test memorization.

Unlike traditional flashcard or spaced repetition applications, Recall Commander does not own the user's knowledge. Users retain complete ownership of their learning materials, which remain in standard Markdown documents that can be edited in any text editor, version controlled with Git, and organized however they prefer.

Recall Commander acts as an intelligent assessment layer on top of those documents. It scans user-defined Question Sources, extracts Recall Commander Question Blocks, generates assessments, evaluates answers, produces detailed review reports, and stores learning history and metadata separately from the knowledge itself.

The philosophy of the project is simple:

> **Knowledge belongs to the user. Assessment belongs to Recall Commander.**

---

# Core Philosophy

Recall Commander is intentionally designed around several principles.

## Ownership

Users own everything they create.

Knowledge, learning materials, assessments, and review reports are stored in open, human-readable formats.

Recall Commander augments those files but never takes ownership of them.

The application should always allow users to access, edit, move, version, back up, or replace their data without vendor lock-in.

---

## Markdown First

Markdown is the primary format used by Recall Commander.

The application should never lock users into a proprietary storage format.

Knowledge should remain:

* human readable
* editable in any text editor
* version controllable using Git
* portable
* future proof

If Recall Commander disappeared tomorrow, every Markdown document would still remain useful.

---

## The Database Stores Metadata Only

The database is **never the source of truth**.

Its responsibility is to store metadata generated by Recall Commander.

Examples include:

* indexed questions
* assessments
* attempts
* scores
* review history
* statistics
* configuration
* future scheduling data

Knowledge always remains inside user-owned Markdown documents.

---

## Assessment Over Memorization

Traditional flashcard applications answer the question:

> "Can you remember this?"

Recall Commander answers:

> "Do you truly understand this?"

The goal is to evaluate the depth of understanding rather than simple recall.

Questions therefore exist at multiple levels of complexity, from simple factual recall to synthesis across multiple concepts.

---

## Question Sources

Recall Commander does not provide a built-in knowledge base.

Instead, users configure one or more **Question Sources**.

A Question Source is simply a directory containing Markdown documents.

Those documents may contain:

* notes
* explanations
* research
* code samples
* diagrams
* references
* Recall Commander Question Blocks

Recall Commander ignores everything except the Question Blocks.

This allows users to continue using their existing workflows and tools without restructuring their knowledge.

Question Sources may include:

* an Obsidian vault
* a Git repository
* a folder of study material
* a collection of personal notes
* a shared community repository
* any directory containing Markdown files

Multiple Question Sources may be configured simultaneously.

Recall Commander recursively scans every configured source and builds a single logical assessment database from all discovered Question Blocks.

---

## Question Blocks

Recall Commander does not interpret Markdown documents as knowledge files.

Instead, it recognizes specially formatted **Recall Commander Question Blocks** embedded inside Markdown documents.

Only these blocks are parsed into assessment questions.

Everything else is ignored.

This allows questions to live naturally alongside notes, explanations, code examples, references, and research without requiring duplicate content or proprietary file formats.

---

## User-Created Knowledge

Recall Commander intentionally separates itself from traditional learning platforms that distribute predefined educational content.

The application provides **the assessment engine**, not the educational material.

Users build their own knowledge while learning from books, courses, documentation, research papers, lectures, experiments, or any other source.

As users learn, they create questions that assess the concepts they consider important.

Knowledge remains entirely user-created and user-owned.

Knowledge sharing is considered a future capability.

Users may choose to share Question Sources with others, allowing communities to exchange question collections and assessment material.

However, this sharing is entirely optional.

Recall Commander itself will never ship with a predefined knowledge repository or curated educational content.

The platform remains completely domain-agnostic.

Its responsibility is not to teach knowledge, but to assess the user's understanding of the knowledge they have chosen to learn.

---

# Goals

The primary goal is to become an intelligent knowledge assessment companion.

Over time Recall Commander should:

* generate assessments
* evaluate answers
* identify weak concepts
* identify misconceptions
* recommend what should be reviewed
* measure long-term progress
* support adaptive scheduling
* encourage deep understanding instead of memorization

Adaptive scheduling is an important long-term goal but is intentionally excluded from MVP 1.

---

# What Recall Commander Is NOT

Recall Commander is **not**:

* a note-taking application
* an Obsidian replacement
* a flashcard manager
* a spaced repetition application
* a document editor
* a knowledge generator
* a learning management system
* a content platform
* a proprietary knowledge store

Its responsibility is assessment.

---

# Guiding Principle

Recall Commander exists to answer one fundamental question:

> **"Do I truly understand what I have learned?"**

Every design decision should support that objective.

Knowledge belongs to the user.

Assessment belongs to Recall Commander.

Everything else is secondary.
